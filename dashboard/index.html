Oberon Guide

Basic workflow

  1. Create a portfolio for the land area and monitoring task.
  2. Add one or more AOIs by drawing on the map or pasting GeoJSON.
  3. Run analysis and wait for the run history to complete.
  4. Review findings on the map and approve, reject, or mark uncertain.

Why the reviews matter

Approved and rejected findings become the local feedback set used to tune thresholds for future runs.

Why Oberon over Copernicus?

  • Automated Change Detection: No manual band math needed. Oberon compares before/after temporal composites automatically.
  • Local AOI Cloud Filtering: Filters cloud coverage locally within your AOI, not the entire Sentinel scene.
  • AI Foundation Models: Optionally use Clay v1.5 multi-spectral embeddings for ML-based disturbance detection.
  • Morphological Calibration: Noise reduction filter calibrated continuously through human feedback.
